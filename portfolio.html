Overview

Problem statement

How do you channel design, code, and personality into a compelling portfolio? This was the question that kept me up at night. An online portfolio shouldn’t be viewed only as a way for clients and employers to contact you but as a place to display my skills and design thinking in an inventive way. Compiling the “right” elements to show was an ordeal itself, but learning and applying front-end skills added an extra level of complexity to the project. The desired outcome was to design a website suitable for a professional in this industry. Additionally, creating a functional site to showcase my work has given me insight into how I may need to communicate with developers in the future.

Context & Challenges

Background & Description

The objective of this project was to design and code a professional web-page to host my design work. The completed online portfolio would allow prospective employers to learn about me and contact me for future opportunities. The project was to be completed in two months. Despite the timeline, the project would be open for further development and improvements.

Problem statement

I’m currently looking for a way to position myself in the UX/UI design industry. To do this, not only did I need to showcase and explain my skills but design a fitting way to do so with my newly learned coding ability.

My solution

A well-organized professional website that displays my current work and helps me implement my new front-end skills.


Process & Insights

Before embarking on the journey of building an online portfolio, I needed to seek help. Developing a professional connection with a mentor through Career Foundry was an important step to completing this project. This mentor offered support and insight while designing and coding my web page.



Infromation & Element Gathering


First, I compiled all the information and the work to showcase. I combined all these elements to create a rough outline of how the website should look. I included the process of researching color schemes, fonts, and specific features I wanted to include during this phase. Black has the psychological effect of feeling formal and sophisticated leading me to make it the main shade for my portfolio. Yellow is a stronger color that can make illustrations and content feel more exciting and capture attention.

Wireframe & Iterations


I was supplied with simple design instructions to follow for the base look of my website. I analyzed these documents similarly to how a designer would analyze a business document and requirement list.


Findings Implementation

Once my sitemap was done and I had the structure in my head, I began to create drafts of how the application would look. I created a few sketches by hand and digitally before committing to one that felt like it made the most sense. The rapid prototyping phase was very flexible. I was able to focus on the "big picture" ideas without getting caught up in details. Below is an example of how user onboarding will be handled within the application.

Implamentation Overview

I implemented a hybrid card sort blending the closed and open card sorts. Through card sorting, I could see how users interacted with the information being considered for my version of this type of application and validate my hypothesis. Users categorized the content in a way logical to them which helped me consolidate my site map.



Conclusion & Reflection

I decided to run a live version of my website through GitHub as I research web hosting and domain options. For me, this is the best environment to further develop and improve my website. The process of learning three coding languages and applying my design skills was grueling, but I appreciated it. I can apply the insights I gained to my future endeavors. Having this website has provided me with an opportunity to build my online presence.
